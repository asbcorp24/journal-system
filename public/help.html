Инструкция по работе с системой электронных журналов

Документ предназначен для супер-администраторов, администраторов, бригадиров и пользователей системы. В инструкции описаны настройка подразделений, справочников, шаблонов журналов, фильтров и работа с удалёнными записями.

1. Назначение системы

Система электронных журналов предназначена для ведения производственных, технологических, лабораторных и иных журналов в электронном виде.

Основные задачи

  • создание шаблонов журналов без программирования;
  • заполнение журналов по ролям и подразделениям;
  • использование справочников со структурированными полями;
  • контроль данных, фильтрация, просмотр и печать.

Для кого подходит

  • производственным подразделениям;
  • бригадирам и ответственным сотрудникам;
  • администраторам, контролирующим данные и доступы.

2. Роли и доступы

Доступ к функциям зависит от роли пользователя, его подразделения и настроек шаблонов. Для администраторов учитывается также иерархия подразделений.

Роль Возможности Ограничения
Супер-админ Полное управление системой, подразделениями, шаблонами справочников, справочниками, журналами и доступами. Нет ограничений в рамках системы.
Администратор Работает со своим подразделением и всеми дочерними. Может редактировать значения справочников, просматривать удалённые записи и восстанавливать их. Не изменяет глобальные настройки супер-админа.
Бригадир Заполняет журналы, просматривает записи своего подразделения, добавляет и редактирует значения доступных справочников. Не удаляет значения справочников и не меняет системные шаблоны.
Пользователь Заполняет доступные журналы и просматривает разрешённые записи. Работает только с доступными журналами.
Если пользователь не видит нужный журнал или справочник, проверьте его роль, подразделение, родительское подразделение и доступы к шаблону журнала.

3. Первичная настройка

  1. Создайте подразделения и при необходимости задайте родительские связи.
  2. Создайте пользователей и назначьте им роли.
  3. Супер-админ настраивает шаблоны справочников.
  4. Создайте справочники и заполните их значениями.
  5. Создайте шаблоны журналов и включите фильтры только у нужных полей.
  6. Проверьте тестовое заполнение журнала и отображение значений из справочников.

4. Подразделения

Подразделения используются для разграничения доступа к пользователям, справочникам, шаблонам и записям журналов. У подразделения можно указать родителя.

Как настроить

  1. Откройте раздел Подразделения.
  2. Создайте новое подразделение или откройте существующее.
  3. Укажите название и при необходимости выберите родительское подразделение.
  4. Сохраните изменения.
Если администратор назначен на родительское подразделение, он получает доступ к журналам и справочникам всех дочерних подразделений.

5. Справочники

Справочник теперь может хранить не одно текстовое значение, а набор полей в формате JSON по шаблону справочника.

Шаблон справочника

Шаблон справочника создаёт супер-админ. В шаблоне доступны поля типов: текст, число, дата, время и список.

  • у каждого поля есть название и системный ключ;
  • поле можно сделать обязательным;
  • поле можно отметить как Уникальное;
  • для поля типа список задаются варианты выбора.

Создание справочника

  1. Откройте раздел Справочники.
  2. Нажмите Добавить справочник.
  3. Введите название.
  4. Выберите шаблон или настройте поля справочника.
  5. Назначьте подразделения, которым доступен справочник.
  6. Сохраните справочник.

Работа со значениями

  • админ и бригадир могут открывать справочник из меню и добавлять значения;
  • админ и бригадир могут редактировать существующие значения;
  • только админ может удалять значения справочника;
  • если для поля включена уникальность, система не даст сохранить дубликат.
Если значение справочника уже используется в журнале, перед удалением проверьте последствия.

6. Шаблоны журналов

Шаблон журнала определяет структуру формы, которую будут заполнять пользователи.

Типы полей

Тип поля Назначение Пример
Строка Свободный текст. Комментарий
Число Числовое значение. Температура
Дата Выбор даты. Дата проверки
Время Выбор времени. Время замера
Список Выбор одного значения из набора. Норма / Не норма
Справочник Выбор значения из справочника. Оборудование
Расчётное поле Автоматический расчёт по формуле. Разница температур

Как настроить шаблон

  1. Откройте раздел Шаблоны журналов.
  2. Нажмите Создать шаблон.
  3. Введите название журнала.
  4. Добавьте нужные поля.
  5. Для каждого поля задайте тип, обязательность и параметры.
  6. Для поля типа справочник выберите сам справочник и укажите, какое поле будет отображаться в списке выбора.
  7. Включите галочку Фильтр только у тех полей, по которым нужен фильтр в журнале.
  8. Назначьте доступные подразделения и сохраните шаблон.
Если у справочника несколько полей, отображаемое поле обязательно. Именно его пользователь будет видеть в выборе и в таблице журнала.

7. Заполнение журнала

Форма журнала строится автоматически на основании шаблона.

Правила заполнения

  • обязательные поля должны быть заполнены перед сохранением;
  • числовые поля принимают только числа;
  • поля даты и времени заполняются через встроенные элементы формы;
  • поля-справочники выбираются из списка с поиском;
  • в коротких списках строка поиска может скрываться, при этом остаётся иконка лупы;
  • расчётные поля не редактируются вручную.

Дополнительные возможности

  • журнал можно открыть кнопкой На весь экран;
  • в полноэкранном режиме доступны кнопки печати и добавления записи;
  • удаление записи работает как мягкое удаление: запись помечается удалённой, но не исчезает из базы;
  • админ может включить показ удалённых записей и восстановить запись кнопкой Восстановить.

8. Поиск и фильтрация

В фильтрах журнала отображаются только те поля, у которых в конструкторе шаблона включена галочка Фильтр.

Что можно фильтровать

  • период;
  • пользователя;
  • подразделение;
  • поля справочников и списков;
  • статус;
  • числовые значения;
  • текст комментария.

Порядок работы

  1. Откройте нужный журнал.
  2. Заполните нужные условия фильтра.
  3. Нажмите Применить.
  4. Для очистки используйте Сбросить.
Для администратора доступен отдельный просмотр удалённых записей, если нужно проверить или восстановить их.

9. Частые вопросы

Почему пользователь не видит журнал?

Проверьте роль пользователя, его подразделение, родительское подразделение и доступы к шаблону журнала.

Почему значение из справочника не отображается?

Проверьте, привязан ли справочник к нужному подразделению и выбрано ли отображаемое поле в настройках поля журнала.

Почему нельзя сохранить значение справочника?

Обычно причина в незаполненном обязательном поле или нарушении уникальности. Если поле отмечено как уникальное, одинаковое значение повторно сохранить нельзя.

Куда пропала удалённая запись журнала?

Запись не удаляется окончательно, а помечается как удалённая. Администратор может включить показ удалённых записей и восстановить её.

10. Чек-листы

Чек-лист администратора

  • созданы подразделения и настроены родительские связи;
  • созданы пользователи и назначены роли;
  • созданы шаблоны справочников;
  • созданы справочники и заполнены значениями;
  • для шаблонов журналов выбраны отображаемые поля справочников;
  • включены только нужные фильтры полей;
  • проверено тестовое заполнение журнала.

Чек-лист пользователя

  • выбран правильный журнал;
  • заполнены обязательные поля;
  • значения из справочников выбраны корректно;
  • ошибки валидации отсутствуют;
  • запись появилась в таблице после сохранения.
